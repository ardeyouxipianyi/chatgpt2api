# 更新日志

## v1.2.0

这是一个面向多人使用和多设备访问的同步版本，重点解决网页画图历史只保存在单台电脑浏览器里的问题，并同步刷新 Windows 绿色版。

### 新增

- 画图页面对话历史改为保存到后端服务器，不再只依赖当前浏览器本地缓存。
- 画图历史按当前登录身份隔离保存，管理员和普通用户密钥各自看到自己的历史。
- 新增画图历史后端接口，支持读取、保存、批量迁移、删除单个对话和清空当前用户历史。
- 备份导入导出新增“Image conversations / 画图对话历史”数据项，可随备份包迁移。

### 优化

- 首次打开画图页时，会把当前浏览器里已有的旧本地历史合并迁移到服务器，避免升级后旧记录直接丢失。
- 同一账号在 A 电脑生成过的画图历史，B 电脑登录同一账号后可以从服务器读取。
- Windows 绿色版配置同步补充画图历史备份项。

## v1.0.1

这是一个针对画图页面滚动与对话节点跳转体验的修复版本。

### 修复

- 修复画图页面在新一轮图片生成处理中时，结果区会持续自动追随最新任务，导致无法通过滚轮查看其他对话轮次的问题。
- 修复生成过程中点击右侧对话节点数字后，页面可能又被处理中任务拉回最新节点的问题。
- 修复右侧对话节点栏滚轮无法稳定带动结果区滚动的问题。

### 优化

- 优化画图页面结果区的自动跟随逻辑：切换会话或新增轮次时仍会自动定位到最新结果，但用户手动滚动或点击旧节点后，会暂停自动跟随。
- 优化右侧对话节点跳转方式，改为更准确的即时定位，避免长对话里跳转不生效或位置不稳定。

## v1.0.0

这是 `ardeyouxipianyi/chatgpt2api` 改版后的第一个正式发布版，基于原项目 `basketikun/chatgpt2api` 继续扩展。这个版本的目标是把 ChatGPT 网页端相关能力整理成更容易自托管、使用和分享的图片创作工具：既保留 OpenAI 兼容接口，也提供网页画图、画布式创作、号池管理、日志排查、数据备份迁移和 Windows 绿色版运行方式。

### 核心能力

- 提供统一入口：网页访问 `http://localhost:3000`，Cherry Studio / OpenAI 兼容接口使用 `http://localhost:3000/v1`。
- 支持 OpenAI 兼容图片生成接口 `POST /v1/images/generations`。
- 支持 OpenAI 兼容图片编辑接口 `POST /v1/images/edits`。
- 支持面向图片场景的 `POST /v1/chat/completions`、`POST /v1/responses` 和 `POST /v1/messages`。
- 支持返回 `b64_json` 与本地图片 URL，方便 Cherry Studio 等客户端直接读取图片结果。
- 支持网页端画图、图片编辑、多图参考编辑、下载结果、重新生成和任务取消。
- 支持管理后台、账号池、日志、设置、图片管理、画布创作等页面。

### 画图页面

- 优化网页画图体验，支持生成任务取消，避免任务卡住后只能等待。
- 优化对话式画图页面滚动体验。
- 新增右侧对话节点跳转按钮，可快速跳到每轮对话位置。
- 支持显示生成张数、完成状态、单图下载、整轮下载和重新生成。
- 优化图片结果加入编辑流程，减少 Cherry Studio 等客户端拿不到图片数据的问题。

### 画布式图片创作

- 新增一级页面“画布”，用于画布式图片创作。
- 支持“提示词 -> 生成图片 -> 图片结果落到画布 -> 选中图片继续编辑 -> 形成分支链路”的工作流。
- 支持在未选中任何节点时直接文生图，并自动把对应节点放入画布。
- 支持选中图片节点后基于上游节点、参考图片和当前提示词继续编辑。
- 支持复制当前编辑节点后生成新的下游结果，便于保留原分支并做多版本尝试。
- 支持继续编辑时调整张数、比例、参考附件和提示词。
- 支持多个结果并排排列，形成清晰分支。
- 支持图片节点复制、删除、重试、下载。
- 支持点击图片放大预览，再次点击关闭；滚轮可按鼠标位置缩放。
- 支持画布拖动，不会因为选中节点后完全失去拖动画布自由度。
- 支持画布保存、恢复和删除。
- 优化节点整理逻辑，让独立分支组左右排列，减少新文生图节点混入旧分支。
- 支持连接线颜色区分，并在选中图片时高亮它的上游连接和上游节点边框。

### 反推提示词

- 新增画布左侧“反推提示词”功能。
- 支持上传图片后调用与生图一致的后端接口，让模型根据图片反推可用于生图的提示词。
- 支持使用当前选中的画布图片作为反推输入。
- 支持展示反推输出结果，并可复制或继续用于创作。
- 支持反推任务取消。
- 支持在设置中修改反推提示词默认指令，并保存到后端全局配置。

### Cherry Studio / OpenAI 兼容接口

- 优化 Cherry Studio 图片生成与图片编辑流程，减少 `AI_InvalidDataContentError`、`AI_NoImageGeneratedError` 等兼容性问题。
- 图片接口统一返回客户端更容易识别的图片结果结构。
- 文生图和图生图请求强制使用独立会话，降低历史对话污染导致上游返回纯文本的概率。
- 当上游只返回文字、没有真正产出图片时，会记录明确失败日志，而不是伪装成成功。
- 日志详情中新增“一键复制日志”按钮，方便把完整单条日志复制出来分析。
- 兼容 `/api/logs` 和 `/api/logs/`，避免因为尾斜杠导致日志页拿到 HTML 而不是 JSON。

### 账号池管理

- 支持多账号导入和管理。
- 支持账号信息、额度和状态刷新。
- 新增一键刷新进度展示，减少大账号池刷新时不可见等待。
- 优化大量账号导入后的后台刷新流程，降低一次性导入几千账号时的等待压力。
- 新增图片失败自动换号能力。
- 支持空结果重试、额度耗尽识别、限流账号冷却。
- 支持配置图片账号并发数，提高多账号场景下的成功率和吞吐。

### 设置、备份与数据迁移

- 设置页支持大类配置项折叠隐藏，让页面更清爽。
- 新增管理员密码修改功能。
- 新增配置数据导出与导入功能。
- 导出备份时弹出选择框，可选择要导出的数据类别。
- 导入备份时先选择文件，再弹出选择框，可选择要导入的数据类别。
- 可选数据包括系统配置、账号池、用户密钥、日志、图片任务、画布项目、注册配置、CPA 配置、Sub2API 配置、账号快照、密钥快照，以及生成图片。
- 导入文件入口增加更明确的选择文件提示。

### 网页与本地运行

- 优化本地开发运行方式，统一公开入口到 `3000` 端口。
- 内部 Next 前端服务仅作为内部服务使用，避免用户误以为需要访问额外端口。
- 优化统一代理稳定性，减少页面刷新或连接中断时代理崩溃。
- 优化部分页面在窄屏宽度下的布局，避免底部编辑框不可见或无法编辑。
- 关闭生产绿色版中的开发浮层按钮影响，普通用户运行时不会看到开发工具入口。

### Windows 绿色版

- 提供 Windows 绿色版压缩包。
- 用户无需提前安装 Docker、Python 或 Node。
- 解压后运行 `start.bat` 即可启动。
- 启动后网页地址为 `http://localhost:3000`。
- Cherry Studio / OpenAI 兼容接口地址为 `http://localhost:3000/v1`。
- 默认管理密码为 `chatgpt2api`，建议首次启动后在设置页修改。
- 绿色版默认使用本地 JSON 存储，避免新电脑缺少 Git 导致无法启动。
- 包内包含运行所需的 Python 运行时、后端依赖和已构建的前端静态文件。

### 修复

- 修复日志接口尾斜杠路径导致日志页为空的问题。
- 修复 Windows 绿色版在未安装 Git 的新电脑上可能因为 GitPython 初始化失败而无法启动的问题。
- 修复图片结果成功生成但客户端解析不到图片时的返回结构问题。
- 修复图片编辑接口中图片内容格式不兼容部分客户端的问题。
- 修复画布新增节点可能生成在旧节点中央、遮挡原节点的问题。
- 修复画布等待生成时部分元素频繁刷新闪动的问题。
- 修复画布中任务处理中时无法继续创建其他任务的问题。
- 修复设置页导入导出选项长期裸露在外导致操作不清晰的问题。
- 修复日志详情缺少快速复制能力的问题。

### 使用提示

- 建议先在网页设置页添加可用账号，再进行图片生成。
- 分享给别人使用时，建议先修改管理员密码，并按需要配置用户密钥。
- Cherry Studio 中接口地址填写 `http://你的IP:3000/v1`。
- 如果通过客户端生图失败，但网页日志显示上游返回了一段普通文字，通常说明上游没有真正调用图片工具，需要调整提示词或换账号重试。
